# TC100 Thermocouple/Thermometer BLE Protocol

## Overview

The TC100 is a BLE-enabled thermocouple/thermometer device that advertises with custom service UUID 8801 and company ID 0x1987. The local name follows the pattern `TC100_XXXX` where XXXX is a hex device identifier that also appears embedded in the manufacturer data payload.

## Identifiers

- **Service UUID:** `8801` (custom, not Bluetooth SIG assigned)
- **Company ID:** `0x1987`
- **Local name pattern:** `TC100_XXXX` (XXXX = hex device ID, e.g., `TC100_52FE`)
- **Device class:** `thermometer`

## BLE Advertisement Format

### Identification

| Signal | Value | Notes |
|--------|-------|-------|
| Service UUID | `8801` | Custom UUID, sometimes appears duplicated |
| Company ID | `0x1987` | Unregistered/unknown manufacturer |
| Local name | `TC100_XXXX` | Device ID suffix matches payload |

### Manufacturer Data Structure

Two formats observed — short (10 bytes) and long (18 bytes, payload repeated):

#### Short Format (10 bytes)

```
87 19 f4 c1 56 06 52 fe 00 26
```

| Offset | Length | Value | Description |
|--------|--------|-------|-------------|
| 0–1 | 2 | `87 19` | Company ID 0x1987 (little-endian) |
| 2–3 | 2 | `f4 c1` | Unknown — possibly sensor reading or config |
| 4–5 | 2 | `56 06` | Unknown — possibly temperature data |
| 6–7 | 2 | `52 fe` | Device ID (matches local name suffix) |
| 8–9 | 2 | `00 26` | Unknown — possibly flags or checksum |

#### Long Format (18 bytes)

```
87 19 f4 c1 56 06 52 fe 00 26 f4 c1 56 06 52 fe 00 26
```

The 8-byte payload `f4 c1 56 06 52 fe 00 26` is repeated. When this format is used, the service UUID `8801` also appears duplicated in the advertisement.

### What We Can Parse from Advertisements

| Field | Source | Notes |
|-------|--------|-------|
| Device presence | service_uuid or local_name | TC100 thermometer nearby |
| Device ID | local_name suffix / mfr data offset 6–7 | Hex identifier |
| Sensor payload | manufacturer_data | May contain temperature (needs further analysis) |

### What We Cannot Parse (requires GATT connection)

- Precise temperature reading interpretation
- Unit setting (°C/°F)
- Min/max/alarm configuration
- Battery level

## Identity Hashing

```
identifier = SHA256("tc100:{mac}")[:16]
```

## Detection Significance

- Indicates a TC100 thermocouple/thermometer device
- Likely used for cooking, HVAC, or lab temperature monitoring
- Continuous BLE advertisement when powered on
